# 바이브코딩 실전 가이드 (NextPlatform)

코딩 지식 없이 AI와 대화하듯 웹앱을 만드는 **바이브코딩(Vibe Coding)**에 대한 실전 가이드 HTML 문서입니다. 기획자·마케터 등 비개발 직군이 **약 30분 안에** 동작하는 프로토타입을 만드는 흐름을 단계별로 정리합니다.

## 이 저장소에서 보는 법

- **GitHub Pages**: 저장소 Settings → Pages에서 소스(보통 `main` / root)를 켜면 `index.html`이 사이트 진입점으로 배포됩니다.
- **로컬**: `index.html`을 브라우저에서 열면 됩니다.

## 문서 주요 내용

### 바이브코딩이란?

- 안드레이 카르파티가 제안한 개념으로, **코드를 직접 쓰지 않고 말로 설명해 AI가 코드를 만들고 실행**하는 방식입니다.
- 문법보다 **요구사항을 명확히 설명하는 능력**이 핵심이라고 강조합니다.

### 기획자가 배워야 하는 이유

1. **아이디어 검증 속도** — 기획·스프린트 대기 대신 짧은 시간에 동작하는 프로토타입으로 압축  
2. **개발자와의 소통** — 직접 만들어보며 난이도·막힘을 체감  
3. **AI 도구 숙련** — 문서 요약만 쓰는 것과 앱 제작에 쓰는 것은 다른 경험

### 실전: 30분 세션 (예시 앱)

**「팀 일일 업무 체크리스트 웹앱」**을 예로 다음 순서를 안내합니다.

| 단계 | 내용 |
|------|------|
| 준비물 | [Claude.ai](https://claude.ai), [Bolt.new](https://bolt.new), 브라우저 (v0.dev, Replit, Lovable 등 대체 가능) |
| STEP 1 | 아이디어를 `[목적][사용자][핵심 기능][참고 UI]` 구조로 정리 (프롬프트 설계) |
| STEP 2 | Claude에서 코드 초안 생성·복사 |
| STEP 3 | Bolt.new에서 즉시 실행·미리보기 |
| STEP 4 | 말로 수정 요청 (**반복적 프롬프팅**) |
| STEP 5 | URL로 배포·공유 |

### 도구 비교

Bolt.new, Lovable, v0.dev, Replit, Cursor, Claude+직접 실행 등 **특징·추천 대상**을 표로 정리합니다.

### 한계와 현실 조언

- **어려운 영역**: 복잡 DB, 고보안(금융·의료), 대규모 동시 접속 등  
- **잘 맞는 영역**: 내부 도구, MVP·프로토타입, 간단한 폼·체크리스트, 데모  
- 바이브코딩은 **완성 제품**이 아니라 **빠른 검증** 도구로 포지셔닝합니다.

### 기획자용 프롬프트 공식

`[역할][목표][기술][UX][조건]` 틀(React, Tailwind, 단일 파일 등)을 제시합니다.

### 마무리·자료

- 본질은 **코딩이 아니라 AI와 협업하는 능력**이라는 메시지  
- **2026 Vibe Coding Playbook** 슬라이드 등 참고·다운로드 링크(문서 본문 내)
